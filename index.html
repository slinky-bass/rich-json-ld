1.1 Introduction

Before looking at the technical aspects of how to create a rich snippet for video, let’s first take a look at what a rich video snippet is and what JSON-LD is and stands for. We shall then go through the basics of creating a rich video snippet with JSON-LD and finally we’ll take a brief look at how the JSON-LD approach to creating structured data differs from marking up structured data with the microdata and RDFa formats and how this comes with some benefits and possible concerns.

1.2 Rich Snippets

Rich snippets are part of Google’s enhanced presentation of search results, which encourage developers (and consequently rewards users) to use their program for marking up certain html content into structured data. By following Google’s Rich Snippet program in particular, certain data categories, such as recipes, events, products and videos, can benefit from being marked up into structured data by being presented as rich snippets in a users search results.

2.1 JSON-LD, an introduction

There are three ways we can mark-up structured data, RDFa, microdata, and the latest format to be accepted by Google, JSON-LD. So what is JSON-LD? JSON stands for JavaScript Object Notation, which is a lightweight format developed for representing data. LD is short for Linked Data, and in the context of the JSON-LD format, linked data means related data. In other words linked data is data that is related to other data.

Let's take the above pictured rich video snippet, the Hobbit, as an example. The movie itself will have its own set of data, marked-up following the common vocabulary set for the videoObject, as defined by schema.org. This vocabulary, or schema, for the videoObject defines a standard set of type names and property names for marking-up structured data, while JSON-LD provides the set of rules for representing the relationships between data in both a human readable way as well as making it easily readable by search engines and their crawlers.

2.2 How to create rich video snippets with JSON-LD

In order to qualify for the rich snippet feature, we first need to make sure we provide certain basic, visible information about the video embedded on our page, namely a title, description, and a working url to the video itself. The html will look something like the below example:

  <div>

    <iframe width="560" height="315" src="https://www.youtube.com/embed/A-kX0Aut-18?list=PLEDEEE61C34CCF962" frameborder="0" allowfullscreen>Sorry, you are unable to watch this video because IE is so old it's fossilized.</iframe> 
            
    <article>

        <h2>Rich Snippets: Product Search</h2>

        <p>Rich Snippets are markup tags that webmasters can put in their sites in order to tell Google what type of content they have on their site so that we can better display it in search results.</p> 
    </article>

  </div>

Once we have all the relevant data visibly displayed on our page, we can then make that data meaningful using a combination of JSON-LD and schema.orgs standardized vocabulary for the videoObject. As mentioned earlier, all marking-up of data with JSON-LD is contained within a <script> block with a type value of “application/ld+json”. The mark-up for our video would look something like this: 

      
<script type="application/ld+json">
{
  "@context": "http://schema.org",
  "@type": "VideoObject",
  "name": "Rich Snippets: Product Search",
  "description": "Rich Snippets are markup tags that webmasters can put in their sites in order to tell Google what type of content they have on their site so that we can better display it in search results.",
  "thumbnailUrl": "http://danielle-eriksen.webportfolio.capetown/rich-snippets/sources/video-thumbnail.jpg",
  "uploadDate": "2011-12-05T08:00:00+08:00",
  "duration": "PT1M11S",
  "contentUrl": "https://www.youtube.com/watch?v=A-kX0Aut-18",
  "embedUrl": "http://www.example.com/videoplayer.swf?video=123",
  "interactionCount": "11124"
}
</script>

Looks pretty simple, right? However, merely structuring the video data on page is not quite enough. Google provides us with a Structured Data Testing Tool to help us verify that we've conformed with their requirements as well as the standardized vocabulary provided by schema.org. It is also recommended that we also provide a video sitemap in addition to our mark-up efforts.

2.3 Video Sitemaps

While this may sound a little intimidating, sitemaps are merely separate xml documents saved into folders of their own along with the rest of the contents of our webpage. A video sitemap describes our video in addition to the mark-up we’ve already done with JSON-LD and schema.org. Sitemaps are separate from our webpages and are submitted directly to Google via Webmaster Tools. By submitting a sitemap for our video we can let Google know about our video directly, rather than having to wait for their crawlers to find and index the page.

Before we can test and submit sitemaps through Googles Webmaster Tools, we first need to verify our site, as well as any additional sites referenced in the sitemap. Once verified, we can then test and ultimately submit our sitemap through the Sitemaps section of Webmaster Tools. Below is a simple example of all the tags required by Google, along with explanations of what kind of description goes with each tag.

Video Sitemap Required Tags

Tag Explanation
 <loc> The URL of the page you’ve embedded the video on
 <video:video> A tag which just wraps around all of the subsequent “<video>” tags for each video referenced per <loc> tag
 <video:thumbnail_loc> The URL of the video thumbnail image, which should be a .png or .jpg file. The image should be 16x9 aspect ratio, ideally either 1920x1080 or 1280x720 pixels
 <video:title> The title of the video - treat this exactly like a page title. Plain text, with keywords and around 70 characters
 <video:description> Treat this exactly like a meta description, but for your video - 160 characters-ish and plain text, including keywords
 <video:content_loc> OR <video:player_loc> content_loc should point to the actual video file URL (.mp4/.mov etc) if you have access to it. Otherwise, such as in instances where you’re embedding a flash player, you can include a player_loc tag which references the URL of the player. You can include both if you wish, but it’s not necessary.You can also include “allow_embed” and “autoplay” elements within the player_loc tag - which indicate whether Google can embed the video directly in the search results (which they currently don’t really do, but may do more in the future) and whether or not clicking through to the page can trigger an autoplay. You don’t really need to worry too much about this, but to future proof the sitemap with the best functionality, I normally include the attributes in there as standard so the tag reads <video:player_loc allow_embed=“yes” autoplay=“ap=1”>PLAYER-URL</video:player_loc>

3.1 Benefits of using JSON-LD

One of the greatest benefits of using the JSON-LD format is that it is incredibly easy to implement on page compared to the microdata and RDFa approaches. All mark-up required to satisfy Google’s rich snippet criteria can be neatly contained within <script> tags, which can be tucked away anywhere on our page, as opposed to having to weave advanced data mark-up with visible content into the HTML in the <body> of our page. This makes it easy for developers to quickly scan through a block of marked-up data and allow us to identify what has been marked-up. The JSON-LD approach is also in line with the principle of “separation of concerns” in that it allows developers to keep HTML tree structures and data structures separate.

Below are two very basic examples from the Google Developers to illustrate the difference in approach between marking up with JSON-LD and microdata

JSON-LD
microdata
  <script type="application/ld+json">
      {
        "@context": "http://schema.org",
        "@type": "VideoObject",
        "name": "Title",
        "description": "Video description",
        "thumbnailUrl": "thumbnail.jpg",
        "uploadDate": "2015-02-05T08:00:00+08:00",
        "duration": "PT1M33S",
        "contentUrl": "http://www.example.com/video123.flv",
        "embedUrl": "http://www.example.com/videoplayer.swf?video=123",
        "interactionCount": "2347"
      }
  </script>
  
  <div itemscope itemtype="http://schema.org/VideoObject">
      <span itemprop="name">Title of video</span>
      <span itemprop="description">Video description</span>
      <img itemprop="thumbnailUrl" src="thumbnail1.jpg" alt="thumbnail text"/>
      <meta itemprop="uploadDate" content="2015-02-05T08:00:00+08:00"/>
      <meta itemprop="duration" content="PT1M33S" />
      <link itemprop="contentUrl" href="http://www.example.com/video123.flv" />
      <link itemprop="embedUrl" href="http://www.example.com/videoplayer.swf?video=123" />
      <meta itemprop="interactionCount" content="2347" />
  </div>

3.2 Concerns with JSON-LD

While in many respects the ability to cleanly separate data mark-up from visible HTML content is a boon, it is also a potential detractor for the very same reasons. As mark-up contained within the <script> tags is invisible to the user, there is a higher chance that content not visible on the page is marked up, which is explicitly against search engine guidelines. In other words, there is the possibility that spammers might take advantage and mark-up invisible data in order to manipulate search engine results.

Another practical concern is that if we have a page that requires lots of mark-up, all the visible content requiring mark-up would need to be repeated in the <script> tags in order to validate properly, potentially leading to slower response times due to all the duplicated data.

4.1 Rich Snippets for Video, why bother?

Video results appear both in Everything search and in Video search. Usually, they will appear as a normal result with some video-specific information, such as a thumbnail and duration. When a user clicks on the video result they will be taken to your page, where they can watch your video.

Video rich snippets have a higher click through rate than regular search results. By giving the user a rich video snippet choice in addition to regular high ranking results, not only are you providing a richer search experience for the user, but you also create a valuable asset for the kind of pages you want to drive a lot of search traffic to, directly from the search results.
